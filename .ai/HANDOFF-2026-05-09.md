## The architecture decision (Option B)

After 2 prior rewrites + adversarial review pass + de-risking spike, three options were on the table:

- **A — In-process `setupInitialTenant`**: rejected because constructing a MikroORM EM under Playwright's loader hits the stage-1-decorators incompatibility (microsoft/playwright#29646) — the EXACT reason the predecessor SPEC-2026-05-09 was abandoned. Don't re-attempt this.
- **B — `mercato test:bootstrap-tenant` CLI subprocess**: chosen. Subprocess escapes Playwright's loader; subcommand passes the discipline-rule Slack-explainability test (legit dev tool for staging seeding / demo provisioning / customer onboarding / DR restores).
- **C — Single shared tenant + per-worker namespacing**: rejected because this spec exists *because* shared-tenant + namespace-discipline already failed (the 33-spec deletion was the proof). Going back to that posture would be choosing the failure mode we're trying to design out.

**Implementation strategy for B:**
1. Interim path while upstream PR is in review: `scripts/bootstrap-test-tenant.ts` in this app wraps `setupInitialTenant` from compiled `@open-mercato/core` dist (no decorator load in test code), invoked via `node:child_process` per worker.
2. After PR #1878 merges + version bump: replace local script invocation with `mercato test:bootstrap-tenant` subcommand. Delete the local script.
3. **60-day exit gate (2026-07-08)**: if PR #1878 doesn't merge by then, the local script becomes permanent. It's still not a test seam (lives in `scripts/`, wraps a real production helper).

## The discipline rule (load-bearing)

> **If OM core wouldn't merge it upstream, we don't add it locally.**

Lives in SPEC-2026-05-09b's "Discipline: OM core is the reference" section. Concretely:

- PRM `__integration__/` specs and fixtures import only from `@playwright/test`, `@open-mercato/core/helpers/integration/*`, and PRM-local helpers at `src/modules/prm/testing/integration/*` and `src/modules/prm/__integration__/fixtures/*`.
- No test-only HTTP routes. No `if (process.env.NODE_ENV === 'test'` or `if (process.env.OM_*_TEST_*` blocks anywhere in `src/modules/**`. No env-var-gated production behavior.
- Any new HTTP route added during test work must pass the **Slack-explainability test**: a support agent on a prod incident, with no test context, must be able to write a one-sentence Slack thread explaining when they would call this route. If they can't, it's a test seam — reject it.
- If a needed capability doesn't exist in OM core: upstream PR > existing OM core pattern > descope. **Add a test seam in PRM is NOT on this list** and is treated at the same severity as a leaked credential.
- **Forward-going Portal-entity coverage rule (Discipline §6)**: any PRM domain entity that surfaces in the customer portal (`[orgSlug]/portal/*` routes) ships with at least one happy-path portal-flow smoke spec in the same PR as the portal pages. Backend-only entities are exempt.

## Process lessons (for future sessions)

1. **Before flipping any spec from DRAFT to READY/APPROVED, dispatch a fresh om-cto adversarial review subagent** (different agent instance, no session context). Same-agent revalidation converges too quickly on the author's own framing. The 2026-05-09 session demonstrated this concretely — 3 HIGH-severity findings landed only on the fresh-agent review pass after multiple in-context validation passes had missed them. Captured as `feedback_pre_flip_adversarial_review.md` in memory.

2. **Adversarial reviews must actually execute** (existing memory rule, reaffirmed): code-only review (greps + reads) misses Playwright/loader/decorator runtime issues. The fresh om-cto review caught the bootstrap gap because it traced `setupInitialTenant` to its callers, NOT just by reading the route handler.

3. **The discipline rule's enforcement surface in this project is `om-code-review` skill** (the project doesn't run CI). For mechanical recurrence prevention, the spec also adds local-only enforcement: prebuild grep gate considered but not adopted (user said no CI; om-code-review checklist is the gate).
