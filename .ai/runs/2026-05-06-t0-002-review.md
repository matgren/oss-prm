### M3 — Wrap tests 1 and 2 of the new spec in `try/finally`

Tests 1 and 2 in `TC-PRM-T0-002-create-flow-ui.spec.ts` perform create + assertions and only invoke cleanup *after* the assertions succeed. If `waitForURL` or any `expect` throws, the freshly-created LicenseDeal is leaked. (Agency cleanup is currently a no-op so the leak is academic for now, but the pattern should match tests 3 and 4 which already model it correctly.)

**Test 1 (Agency UI flow) — change lines 30-57:**

```typescript
test('Agency create UI flow reaches detail page (no loading hang)', async ({ page, request }) => {
  await login(page, 'admin')

  const uniqueSuffix = Date.now().toString(36)
  const agencyName = `Smoke Agency UI ${uniqueSuffix}`
  const agencySlug = `smoke-ui-${uniqueSuffix}`

  await page.goto('/backend/prm/new')
  await expect(page.getByRole('heading', { name: /create agency/i })).toBeVisible()

  await page.locator('[data-crud-field-id="name"] input').fill(agencyName)
  await page.locator('[data-crud-field-id="slug"] input').fill(agencySlug)
  await page.locator('[data-crud-field-id="tier"] select').selectOption('om_agency')
  await page.locator('[data-crud-field-id="headquartersCountry"] input').fill('US')
  await page.getByRole('button', { name: 'Create' }).first().click()

  await page.waitForURL(/\/backend\/prm\/[0-9a-f-]{36}$/, { timeout: 15_000 })

  const detailUrl = page.url()
  const agencyId = detailUrl.split('/').pop() ?? ''

  try {
    await expect(page.getByRole('button', { name: /^profile$/i })).toBeVisible({ timeout: 10_000 })
    await expect(page.getByRole('heading', { name: agencyName })).toBeVisible()
  } finally {
    const token = await getAuthToken(request, 'admin')
    await deleteAgencyIfExists(request, token, agencyId)
  }
})
```

**Test 2 (LicenseDeal UI flow) — same shape**: extract `licenseDealId` after `waitForURL`, then wrap the visibility assertions + diagnostic dump in `try/finally` so `deleteLicenseDealIfExists` runs unconditionally. Keep the existing diagnostic `try/catch` *inside* the new outer `try` block so the dump still fires on visibility failure.

Optional: drop the redundant `expect(agencyId).toMatch(/^[0-9a-f-]{36}$/)` and `expect(licenseDealId).toMatch(...)` — `waitForURL`'s regex already enforces UUID shape (this is L3 from the review, marked optional).
