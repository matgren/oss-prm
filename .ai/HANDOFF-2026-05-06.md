## Non-negotiable lessons from T0-T2 reviews

Bake these in from commit 1 of any new spec — they were CRITICAL/HIGH blockers in T0's first attempt:

1. **Encryption helpers** on every tenant-scoped query: `findWithDecryption` / `findOneWithDecryption` / `findAndCountWithDecryption` from `@open-mercato/shared/lib/encryption/find`. NEVER raw `em.find` / `em.findOne` / `em.findAndCount` on `prm_*` tables. (T0's first attempt had 17 violations.)
2. **`safeEmit`** for ALL event emissions: `src/modules/prm/lib/safeEmit.ts`. NEVER `.catch(() => undefined)` on event emit. (T0 had 14 silent catches.)
3. **`ColumnDef<RowType>[]`** for DataTable columns. NEVER `columns as any` or `({ row }: any)` cells.
4. **Form initialValues typed** against the form's value type. NEVER `as any` on defaults.
5. **Migration scope discipline**: every generated migration MUST only touch `prm_*` tables you intend to add. After `yarn mercato db generate`, run `wc -l` and `grep -ohE 'table "[a-z_]+"'` on the migration file to verify scope. (T0's first attempt had a 1221-line cross-module migration.)
6. **`/data/` directory**: gitignore was fixed at root level (`/data/` rule). Module-level `data/` directories track normally now.
7. **Spec §11 changelog accuracy**: every claimed file MUST be in `git ls-tree -r HEAD`. Cross-check before reporting done.
8. **Tenant isolation** in every query.
9. **DataTable wires pagination**: `page`, `pageSize`, `total`, `totalPages`, `onPageChange`, `onPageSizeChange`.
10. **All API routes export `openApi` + `metadata`**.
11. **OM primitives only** in UI: `Button`, `IconButton`, `Input`, `Textarea`, `LoadingMessage`, `ErrorMessage`, `FormHeader`, `FormFooter`, `flash`. NEVER raw `<button>`, `<input>`, `<textarea>`. (Discovered late in T2 via DS-guardian; fixed with T0/T1/T2 fix-forward series. New work shouldn't repeat.)
12. **Saga discipline (T2-specific, applies if you write more sagas)**: use platform `WorkflowDefinition` + `WorkflowEventTrigger`. NO custom orchestrator. NO PRM-owned `processed_events` dedupe table. Idempotency via platform's `correlation_key` index + `maxConcurrentInstances=1`.
